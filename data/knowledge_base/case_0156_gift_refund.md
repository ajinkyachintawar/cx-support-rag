# Resolved case — refund request from gift recipient, not purchaser

Case ID: CASE-0156

Situation: A customer contacted support wanting to refund an item that was
purchased for them as a gift by someone else. The recipient did not have
the original order number or the purchaser's account details.

Resolution: Standard refund policy (REF-STD-014 / REF-PREM-030) requires
refunds to go back to the original payment method, which the recipient
does not control. The recipient was offered store credit instead of a cash
refund, valued at the item's current listed price, valid for 12 months.
This is a standing exception process for gift recipients specifically — it
does not apply to non-gift refund requests, which always require purchaser
verification.

Note for future agents: do not require the recipient to track down the
purchaser. Ask only for the order confirmation email if they have access
to it (often forwarded by the purchaser), otherwise proceed on item
description and approximate purchase date.
