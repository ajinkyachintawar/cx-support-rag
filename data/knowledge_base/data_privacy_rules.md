# Data privacy handling rules

Policy code: DPR-001

Any customer request involving the following must be escalated immediately
per ESC-002, never handled directly by a first-line agent:

- Request to delete all personal data ("right to erasure")
- Request to export or access all data held about them
- Suspected data breach or unauthorized account access
- Request to know what data is shared with third parties

First-line agents must NOT confirm, deny, or describe what specific data is
held on a customer beyond what is needed to verify identity for the current
support interaction. Do not read account details aloud beyond order history
relevant to the current issue.

All such requests must be logged with ticket tag "DPR" and routed to the
data protection team, not the standard escalation queue. Response SLA for
DPR-tagged tickets is 72 hours per regulatory requirement, longer than the
standard 4-hour escalation SLA.
